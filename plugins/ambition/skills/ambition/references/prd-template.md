# PRD 完整模板（12 节）

DEFINE 阶段的标准产出格式。先看裁剪指引决定写哪几节，再逐节填写。

## 按深度裁剪

| 深度 | 包含的节 |
|------|---------|
| quick | 1、4、5、7（精简 4 节） |
| standard | 1、2、4、5、6、7、8、12（标准 8 节） |
| full / enterprise | 全部 12 节 |

## 写作要点

- **故事 ID 是全局锚点**：BREAKDOWN 的 Story、PROTOTYPE 的走查、测试用例都引用 US-xxx，编号一旦发出不要复用。
- **Won't 比 Must 更难写也更有价值**：用户最容易在这里和你产生分歧，写具体（"不做多语言"而不是"不做国际化相关功能"）。
- **AC 写可观察的行为**，不写实现（"跳转到主页并显示用户名" ✅，"调用 redirect 函数" ❌）。
- **每个 Must 故事必须有异常场景的 AC**——happy path 人人会写，异常处理才是质量分水岭。
- **从 discovery.md 的 assumption log 继承**：未确认的假设要么在 PRD 评审时确认掉，要么写进第 10 节风险。
